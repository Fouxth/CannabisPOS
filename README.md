# 🌿 CannabisPOS - ระบบจัดการหน้าร้าน (POS) และบริหารจัดการสต็อกครบวงจร

ระบบบริหารจัดการร้านค้า (Point of Sale) ที่ออกแบบมาอย่างพิถีพิถันเพื่อธุรกิจสมัยใหม่ รองรับการขยายสาขาระบบ Multi-tenant พร้อมฟีเจอร์แจ้งเตือนแบบ Real-time และการวิเคราะห์ข้อมูลขั้นสูง

---

## ✨ คุณสมบัติเด่น (Core Features)

### 🛒 ระบบขายหน้าร้านสมัยใหม่ (Advanced POS)
- **หน้าจอขายที่รวดเร็ว**: รองรับการค้นหาสินค้าด้วยระบบ Smart Search และการสแกนบาร์โค้ด
- **ระบบพักบิล (Suspended Bills)**: สามารถเปิดค้างหลายบิลพร้อมกัน เพื่อสลับการให้บริการลูกค้าได้ทันที
- **ส่วนลดและโปรโมชั่น**: คำนวณส่วนลดท้ายบิล หรือเลือกใช้โปรโมชั่นที่ตั้งค่าไว้ล่วงหน้า
- **พิมพ์ใบเสร็จ (Receipt Printing)**: รองรับการพิมพ์ใบเสร็จอย่างย่อ พร้อมปรับแต่งหัวบิล/ท้ายบิลได้เอง
- **คีย์ลัด (Keyboard Shortcuts)**: เพิ่มความไวในการทำงานด้วยปุ่มลัดสำหรับฟังก์ชันหลัก

### 📦 การจัดการสต็อกและคลังสินค้า (Inventory & Stock)
- **ระบบตัดสต็อกอัตโนมัติ**: ตัดสินค้าทันทีเมื่อปิดการขาย พร้อมระบบตรวจสอบสต็อกคงเหลือ
- **Stock Movement**: บันทึกทุกความเคลื่อนไหว (รับเข้า, ปรับปรุง, ความเสียหาย) พร้อมระบุตัวตนผู้ดำเนินการ
- **Low Stock Alerts**: แจ้งเตือนเมื่อสินค้าใกล้หมดสต็อกแบบ Real-time
- **Bulk Import**: รองรับการนำเข้าข้อมูลสินค้าจำนวนมากผ่านไฟล์ Excel

### 🏢 ระบบบริหารจัดการหลายสาขา (Multi-tenant Infrastructure)
- **Central Admin Panel**: หน้าจัดการระบบสำหรับเจ้าของที่ต้องการเปิดหลายสาขา (Manage Tenants)
- **แยกฐานข้อมูล (Database Isolation)**: แต่ละสาขามีฐานข้อมูลแยกจากกัน เพื่อความปลอดภัยและความเสถียรของข้อมูล
- **Dashboard สาขา**: สถิติแยกตามสาขาแบบ Real-time (ยอดขาย, กำไร, จำนวนออเดอร์)

### 🔔 ระบบแจ้งเตือนอัจฉริยะ (Notification System)
- **Real-time In-app Notifications**: แจ้งเตือนภายในเว็บบราวเซอร์ทันทีเมื่อมีกิจกรรมสำคัญ (ผ่าน Socket.io)
- **LINE Flex Message**: ส่งข้อความสรุปยอดขายรายวัน/รายเดือน หรือแจ้งเตือนยอดขายถึงเป้าหมาย เข้าทาง LINE เจ้าของร้านโดยตรง
- **Sales Target Tracking**: ตั้งเป้าหมายยอดขายรายวัน และรับการแจ้งเตือนทันทีเมื่อยอดขายถึงเป้า

### 📈 รายงานและบทวิเคราะห์ (Advanced Analytics)
- **Dashboard สรุปภาพรวม**: กราฟแสดงสถิติลายชั่วโมง, รายวัน และรายเดือน
- **รายงานการเงิน**: สรุปยอดขาย, กำไรสะสม, และรายจ่าย (Expenses Management)
- **Export Reports**: ส่งออกข้อมูลเป็น Excel หรือ PDF เพื่อใช้ในการทำบัญชี

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

### 🎨 Frontend
- **React 18 + TypeScript** (Vite)
- **Tailwind CSS + shadcn/ui**: ดีไซน์ที่สวยงาม ทันสมัย และเป็นมืออาชีพ
- **TanStack Query (React Query)**: การจัดการ Data Fetching และ Caching
- **Zustand**: State Management ที่เบาและรวดเร็ว
- **Recharts**: การแสดงผลกราฟสถิติที่สวยงาม
- **Micro-interactions & UX Optimization**: ปรับแต่งการตอบสนองปุ่มด้วยระบบ `active:scale-[0.97]` (Physical Click Feel) และการหน่วงเวลาเคลื่อนไหวด้วย Custom Cubic-Bezier Transitions 
- **Performance & Accessibility**: ใช้ CSS Layout Containment (`contain-layout`) สำหรับลดการวาดหน้าจอ (Repaint), รองรับ Reduced Motion เพื่อผู้พิการ และลบอาการปุ่มค้าง (Sticky Hover) บนระบบสัมผัส (Mobile/Tablet)

### ⚙️ Backend
- **Node.js + Express + TypeScript**
- **Prisma ORM**: การสื่อสารกับฐานข้อมูลที่มีระบบ Type Safe
- **PostgreSQL**: ฐานข้อมูลหลักที่รองรับโครงสร้าง Multi-tenant
- **Socket.io**: เครื่องมือสำหรับการสื่อสารแบบ Real-time
- **LINE Messaging API**: เชื่อมต่อระบบแจ้งเตือนเข้ากับแอปพลิเคชัน LINE

## 🏗️ โครงสร้างระบบ (Architecture)

ระบบถูกออกแบบให้มีความยืดหยุ่นสูง (Scalable):
- **Middleware Infrastructure**: ระบบตรวจสอบความถูกต้อง (Auth) และการระบุ Tenant อัตโนมัติจาก Domain หรือ User Token
- **Service Layer**: แยก Logic การทำงานชัดเจน (SmsService, SocketService, TenantManager)
- **Audit Logging**: ระบบบันทึกกิจกรรมพนักงาน (ใคร ทำอะไร ที่ไหน เมื่อไหร่) เพื่อความโปร่งใส

## 💻 วิธีการรันโปรเจกต์ (Getting Started)

### 1. ตั้งค่าฐานข้อมูลและ Backend
```bash
cd server
npm install
# คัดลอก .env และตั้งค่า DATABASE_URL
cp .env.example .env 
# สร้าง Schema ฐานข้อมูล
npx prisma generate
npx prisma migrate dev
# รัน Server
npm run dev
```

### 2. ตั้งค่า Frontend
```bash
cd ..
npm install
npm run dev
```

## 🔐 สิทธิ์การใช้งาน (Roles & Permissions)

- **Owner**: เข้าถึงได้ทุกฟังก์ชัน จัดการตั้งค่าร้านและพนักงาน
- **Admin/Manager**: จัดการสินค้า สต็อก และดูรายงานสำคัญ
- **Cashier**: เน้นการขายหน้าร้าน และจัดการบิลที่ตนเองรับผิดชอบ
- **Viewer**: ดูข้อมูลและรายงานได้อย่างเดียว ไม่สามารถแก้ไขได้
